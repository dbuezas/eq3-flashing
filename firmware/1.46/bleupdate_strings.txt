EQ3 CC-RT-BLE Firmware v1.46 (bleupdate.bin) — Extracted Debug Strings
======================================================================

The firmware is a WICED Smart SPAR patch (not a flat binary), consisting of
multiple records loaded to different SRAM addresses. Full decompilation
requires a tool that understands the SPAR format (e.g., Ghidra with WICED plugin).

The debug strings below reveal the complete firmware architecture:

=== NOTABLE FINDINGS ===
1. "Transmission of Internals detected" — MCU sends internal data to BLE chip.
   This is the mystery data that might contain temperature readings.
   The BLE chip receives it but we don't know if/how it's exposed.

2. "((attrPtr[1]&0xc0) >> 6)" — Command type is extracted from byte 1 bits 7:6.
   This suggests the command byte format has a type field in the upper 2 bits.

3. "FD+CC received from BCM2, scanmode may start" — The MCU can trigger BLE
   scanning. 0xFD is the UART frame start, 0xCC is a specific message type.

4. "Activate Testmode" + "Sent 0xFD+0xFF" — Factory test mode uses UART
   command 0xFF. "0xFD+0xFE" is another test command.

5. "PartyFlag Active" / "Data Read Flag Active" — Unknown flags, possibly
   related to multi-device coordination or data readout mode.

6. No calibration/AdA command — Motor calibration is entirely MCU-side.
   The BLE chip just relays commands and has no motor awareness.
